# PRAWNHUB History Page Debug (Android Side)

## Issue

ESP32 is successfully writing history records to:

```text
/ShrimpHub/history
    -AutoGeneratedKey
        timestamp
        temperature
        tds
        turbidity
        waterLevel
```

Firebase contains history data, but the Farmer History page shows no records, empty charts, or date filtering returns no results.

## Check Android App

### Verify Firebase Path

Confirm HistoryActivity reads:

```java
FirebaseDatabase.getInstance()
    .getReference("ShrimpHub")
    .child("history");
```

and NOT:

```java
history
ShrimpHub/history/history
History
```

---

### Verify Timestamp Parsing

Current Firebase format:

```json
{
  "timestamp": 1778686057,
  "temperature": 31.06,
  "tds": 0,
  "turbidity": 53.99,
  "waterLevel": -1
}
```

Ensure Android correctly converts Unix timestamps:

```java
1778686057 -> milliseconds
1778686057000
```

before date filtering.

---

### Verify Date Range Filter

Check:

```java
isInSelectedDateRange()
```

Make sure records are not excluded because:

- Start date > End date
- Wrong timezone
- Wrong timestamp conversion
- RTC timestamps interpreted incorrectly

Temporarily bypass filtering:

```java
return true;
```

If records appear, filter logic is the problem.

---

### Verify History Parsing

Add logs:

```java
Log.d("HISTORY_DEBUG", child.getValue().toString());
Log.d("HISTORY_DEBUG", "Timestamp=" + timestamp);
```

Expected:

```text
Timestamp=1778686057
Temperature=31.06
TDS=0
Turbidity=53.99
WaterLevel=-1
```

---

### Verify RecyclerView

After Firebase load:

```java
Log.d("HISTORY_COUNT", String.valueOf(allItems.size()));
```

Expected:

```text
HISTORY_COUNT > 0
```

If Firebase has data but count is 0, parsing is failing.

---

### Verify Charts

Add:

```java
Log.d("CHART_COUNT",
String.valueOf(temperatureEntries.size()));
```

Expected:

```text
CHART_COUNT > 0
```

If zero, timestamps are not being parsed correctly.

---

## Expected Working Firebase Structure

```text
ShrimpHub
 └─ history
     └─ -AutoKey
          timestamp: 1778686057
          temperature: 31.06
          tds: 0
          turbidity: 53.99
          waterLevel: -1
```

No additional history node required.

## Goal

Find why Android HistoryActivity is not displaying valid Firebase history records even though ESP32 successfully writes them.
